Part A – Understanding the CIA Triad

1. Confidentiality

Definition
Confidentiality means ensuring that information is accessible only to authorized people.

Importance
Protects sensitive information from unauthorized access.

Example
Only students and faculty can access college examination records.

2. Integrity

Definition
Integrity means ensuring data remains accurate and unaltered.

Importance
Prevents unauthorized modification of information.

Example
Student marks should not be changed by unauthorized users.

3. Availability

Definition
Availability means information and systems are accessible when needed.

Importance
Ensures services remain operational.

Example
College website should be accessible during admission periods.

Part B – Real World Case Study

WannaCry Ransomware Attack (2017)

What Happened?
WannaCry was a ransomware attack that infected over 200,000 computers across 150 countries. It encrypted files and demanded payment in Bitcoin.

CIA Triad Affected
Availability (majorly)
Integrity (partially)

Impact
Systems became unusable.
Hospitals, businesses, and government organizations were affected.
Millions of dollars in damages.

Prevention
Regular system updates.
Security patches.
Backups.
Antivirus software.

Part D – Threat Identification

1. Insider Threat

Description
Threat caused by employees or trusted individuals.

Impact
Data theft and misuse.

Prevention
Access controls and monitoring.

2. Malware

Description
Malicious software that damages systems.

Impact
Data loss and system compromise.

Prevention
Antivirus and updates.

3. Phishing

Description
Fraudulent attempts to steal information.

Impact
Credential theft.

Prevention
User awareness and MFA.

4. Weak Passwords

Description
Simple passwords that are easy to guess.

Impact
Unauthorized account access.

Prevention
Strong password policies.

5. Unpatched Systems

Description
Systems missing security updates.

Impact
Exploitation of vulnerabilities.

Prevention
Regular patch management.

Part F – Mini Security Consultant Activity

Cybersecurity Improvement Report for College

Top 5 Risks

Data breaches involving student records.
Phishing attacks targeting staff and students.
Malware infections.
Weak password practices.
Unpatched software vulnerabilities.

Top 5 Recommendations

Implement Multi-Factor Authentication.
Enforce strong password policies.
Conduct regular security awareness training.
Keep all systems updated.
Perform regular backups and security audits.
Expected Benefits
Improved protection of sensitive information.
Reduced risk of cyber attacks.
Faster recovery from incidents.
Increased trust among students and staff.
